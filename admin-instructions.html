The honest bit. Nothing can place a widget inside the Claude Code app's left sidebar — that part of the app isn't extendable by any plugin. This is the faithful workaround: a separate floating chip that stays on top of every window and that you drag to wherever you want it. Visually it can sit right alongside the app; technically it's its own little window.

1Requirements

  • Windows 10 or 11. Uses built-in Windows PowerShell + .NET WinForms — nothing to install.
  • Claude Code, used at least once. The widget reads the local session transcripts Claude Code writes under %USERPROFILE%\.claude\projects\. It never touches the network.

2Install & run

Easiest — one click. Unzip the folder, then double-click Install.cmd. It copies the widget into place, clears the “downloaded from the internet” flag so nothing gets blocked, adds a Desktop shortcut with a matching icon, and launches it. If Windows shows a “Windows protected your PC” box, click More info → Run anyway — it's a small, readable script.

Or run it in place. Prefer not to install? Just double-click Start Widget.vbs. The panel appears at the bottom-left with no console window. Drag it wherever you want — it remembers the spot.

It works immediately if you've used Claude Code before — it reads your existing session history. If it says “no Claude usage yet today,” just send one message in Claude Code and it populates.

3Keeping it running day to day

The widget keeps running until you close it or restart the PC. It survives sleep, so if you just leave your machine on, it'll still be there when you come back — nothing to do.

  • Bring it back any time: double-click Start Widget.vbs again — exactly the same as launching it the first time.
  • Never think about it again: double-click Add to Startup.cmd once. From then on it launches automatically every time you sign in to Windows — so the morning after a shutdown or reboot, you log in and it's already there. Remove from Startup.cmd undoes it.
Coming back the next day? If you shut down or restarted, log in and — if you ran Add to Startup.cmd — it's already running. If not, double-click Start Widget.vbs. Auto-start only kicks in at sign-in, so if you close the widget in the middle of a session it won't reappear on its own until your next login; Start Widget.vbs brings it right back.

4Move, history & close

  • Move: click and drag anywhere on the panel.
  • History calendar: click the calendar button in the top-right (or right-click → History (calendar)). It opens a per-day usage calendar in your browser — a heat-mapped month grid, summary cards (all-time spend, busiest day, averages, total tokens), a “by project” breakdown (which project used how many tokens / how much), a Cost/Tokens toggle, and hover-for-details. Your history is saved locally, so it keeps growing even after Claude Code prunes old transcripts.
  • Refresh: click the ↻ button to re-scan right now.
  • Close: click the × in the top-right, or right-click the panel → Exit.
  • Right-click menu: History (calendar) Refresh now Open instructions Exit

5What the numbers mean

title
The label “Claude usage,” with a ↻ refresh button and a × close button.
spent today · cached
The headline number: today's cache-aware cost, summed across every session since midnight. The realistic API estimate — see the note below.
if billed per token
The raw “sticker” price for today: every input, cache, and output token at full list rate, no cache discount. Always larger — usually several times the cached figure.
tokens
Today's cumulative tokens (input + output + cache) as a bar against a daily budget you can set (default 2B). A “how heavy is today” gauge. Note: this is a tunable budget, not your real plan limit — see the note below.
model rows
One row per model family you've used today (Opus, Sonnet, Haiku, Fable, …), summed across all sessions. Each shows that model's cache-aware cost and ↓ the tokens it generated. The panel grows/shrinks to fit — no empty rows.
output · turns · sessions
Today's total tokens generated, the assistant-turn count, and how many distinct sessions contributed today.
updated … ago
How fresh the reading is — the time since the most recent turn in any of today's sessions.
About the dollar figures. Both are estimates at API list prices. If you're on a Claude Max/Pro subscription you don't pay per token — your plan covers it — so read them as “what today would have cost on the pay-as-you-go API,” a sense of weight, not a bill. Spent today · cached is the realistic number (caching is part of real API billing); if billed per token shows the raw token volume's sticker price with the caching discount removed. Both include work done by any sub-agents your sessions spawned (Explore/Task/parallel helpers).
About the “tokens” bar. It's today's cumulative tokens against a daily budget you choose ($DailyBudgetTokens, default 2B) — a sense of how heavy today has gotten. It is not your Claude plan's real rate-limit: that number only exists in the data Claude Code feeds its status line, which this standalone widget never receives. So treat the bar as a tunable gauge, and set the budget to whatever daily ceiling is meaningful to you.

6How it works (under the hood)

Every turn, Claude Code appends your conversation — including exact token counts and a timestamp — to a transcript file under %USERPROFILE%\.claude\projects\. The widget looks at every transcript touched today (and any subagents\*.jsonl beside them), and for each one reads only the bytes appended since it last looked, keeping a per-file running total of the turns timestamped today. It sums those into one daily figure and buckets cost + output by model family. From that it paints the headline cost, the daily-tokens bar, one row per model used today, the raw cost, and the footer. Because it only re-reads new bytes, even a busy live session refreshes in a few milliseconds, and at local midnight the counters reset. No status line, no background service, no network — just one small program reading files that already exist.

7Tweaking it

Everything lives in usage-widget.ps1 — open it in any text editor:

  • Daily-tokens budget: $DailyBudgetTokens (default 2000000000 = 2B) is what the tokens bar is measured against. Lower it if you want the bar more sensitive.
  • Prices: the Get-Price function holds per-model list rates — edit if Anthropic's prices change or you want different numbers. The cache-aware cost derives cache rates from the input rate; the per-token cost uses the input rate flat.
  • Row colours / order: $FamColor and $FamOrder. Every family with usage today is shown automatically — you don't choose which.

8Troubleshooting

  • “no Claude usage yet today”: you haven't used Claude Code yet today, or it's just past midnight. Send a message and it appears.
  • Numbers feel frozen: they only change when a new turn is written. The “updated … ago” line tells you how fresh it is.
  • The cost looks big: that's the API-list estimate for the whole day, not a charge — see section 5. The “per token” figure is deliberately the larger, no-caching number.
  • tokens bar nearly full: you've had a heavy day; raise $DailyBudgetTokens if you want more headroom.
  • Only one model row: you only used one model today. Rows appear as you (or sub-agents) use more — e.g. Haiku often shows up once a sub-agent runs.
  • Widget disappeared: double-click Start Widget.vbs again. Only one copy runs at a time, so you can't accidentally stack duplicates.
  • Windows blocked the .vbs / .cmd: right-click the file → Properties → tick Unblock (appears on files downloaded from the internet), then run it.
