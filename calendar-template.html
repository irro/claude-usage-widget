Known limitations
What this tool can't track.
  • Claude Design. Anthropic doesn't release token counts for it — there's nothing to read, on or off your machine.
  • claude.ai chat & co-work (the web app). That runs entirely on Anthropic's servers; nothing is ever written to your machine for it.
  • Reasoning effort & fast mode. Not recorded in the local transcript format Claude Code writes, so there's no field to read.
Only Claude Code / Claude CLI usage is tracked (local transcripts under %USERPROFILE%\.claude\projects\) — including a local model like Ollama if you ever point Claude Code at one (shown as "Local", always $0).

Usage windows
Anthropic doesn't publish exactly how many tokens a 5-hour / weekly / Fable window gives you, and that could change without notice. This tracks your own observed usage in each window over time so you can spot a pattern — it is not a plan quota. Reasoning-effort, fast-mode, and Claude Design usage aren't included: none of that is recorded anywhere this tool can read. Day tokens and Trailing 7d tokens below are Claude/Cloud only (Local/Ollama usage is never mixed in, since it's a different tokenizer); Peak 5h tokens is combined across everything, since hourly detail isn't split by model.

Weekly usage
Compute weeks reset every Tuesday at 1:00 PM, mirroring Claude's own weekly limit reset — not calendar (Sun–Sat) weeks. The boundary Tuesday is split to the hour where the underlying data allows it; older days that predate this feature (or were pruned before a rescan) fall back to whole-day attribution.
